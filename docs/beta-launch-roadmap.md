# Beta Launch Roadmap (4 Weeks)

## Strategy lock

- Audience: OSS maintainers
- Distribution: GitHub-first
- North-star KPI: external repos using `@v1`
- Capacity model: solo founder
- Outbound style: targeted PRs/issues
- Broad launch gate: >=95% self-test pass rate over rolling 7 days

## Week 1: Foundation

- Lock positioning and CTA in docs/README.
- Ship support policy and issue templates.
- Start weekly metrics collection files.
- Record reliability baseline.

Exit criteria:

- Support policy published.
- Metrics process documented and runnable by one person.

## Week 2: Early adopter wave

- Build target list of 30 repos.
- Run targeted setup PR/issues.
- Track merge outcomes and blockers.
- Apply same-week docs/CLI friction fixes.

Exit criteria:

- First 8-12 external repos on `@v1`.
- Top setup blockers documented and prioritized.

## Week 3: Distribution expansion

- Prepare and publish marketplace-facing copy/checklist.
- Link proof repos and before/after CI examples.
- Share adoption evidence, not generic claims.

Exit criteria:

- Marketplace positioning live.
- Proof assets linked and green.
- External adoption passes ~15 repos.

## Week 4: Beta push + stabilization

- Announce beta with proof links and adoption count.
- Enforce reliability gate before broad outreach.
- Publish beta learnings and next-30-day plan.

Exit criteria:

- 15-25 adopted external repos.
- Reliability gate maintained.
- Post-beta roadmap published.
